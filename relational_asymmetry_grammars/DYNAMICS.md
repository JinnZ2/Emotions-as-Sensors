# Dynamics catalog

Three dynamics, in the stripped format. Each is observable-only, carries at
least one falsifier, and passes the attribution linter. Work in progress.

A shared vocabulary is used across all three:

| Term | Definition (observable, non-mentalistic) |
|---|---|
| **Reset** | The discrete event that re-arms the loop and returns the system to its recurring configuration. |
| **Response-contingent** | A reset whose trigger is strictly the observer's preceding action (not time, not external stress). |
| **Autonomous** | A reset keyed to time or external stress, not to the observer's action. |
| **Restored state** | The configuration the system returns to after a reset. Named by what recurs, not by what anyone wants. |
| **Accretion / ratchet** | The threshold marker accumulates across cycles and does not decay. |
| **Load** | A measurable prohibition or offloaded task that lands on one party. Discrete (step) or continuous (duration-scaled). |
| **Externalized cost** | A load that lands on one ledger and never appears on the other. Unmetered. |
| **Threshold marker** | The accommodation level treated as baseline in a given cycle. |
| **Pre-coding** | A frame in which an action is classified as an offense before its content is evaluated. |

---

## Dynamic 1 — Recruited Justification

**Signature.** A stated reason for an outcome fails when the same reason is
applied to a structurally parallel case.

**Rules.**
- R1: An outcome is fixed → a reason is supplied after the outcome. (Order is
  observable: the reason appears in defense of an already-fixed outcome.)
- R2: The supplied reason is applied to a parallel case → the reason does not
  hold there. (The reason does not carry load; it is not stable across cases.)

**Load.** *Parallel-scrutiny labor* (discrete): the observer carries the work of
testing the stated reason against parallel cases. The other party does not.

**Falsifier.** F1 (parallel-case): the stated reason holds consistently across
all structurally parallel cases tested.

## Dynamic 2 — Self-Sealing Reframe

**Signature.** Naming a pattern is responded to by reclassifying the act of
naming as hostility; any following response is read as confirmation of the
hostility.

**Rules.**
- R1: A pattern is named → the naming is reclassified as hostility. (Topic
  shifts from the content named to the act of naming.)
- R2: Any response is given to the reclassification → the response is read as
  confirming the hostility. (Escalation, clarification, and silence are read
  identically.)
- R3: The premise is stated → it is consistent with every possible response.
  (Zero information content: a premise compatible with all outcomes
  distinguishes none.)

**Loads.**
- *Burden-of-proof transfer* (discrete): the observer carries the burden of
  disproving an ascribed hostility never asserted by the observer.
- *Reframe-absorption* (continuous): the observer holds the reframe over time;
  the content of the original observation stays unexamined for the duration.

**Falsifier.** F1 (content-evaluation): a counter-observation is ever evaluated
on its content rather than reclassified as hostility.

**Note on the exit.** No move wins from inside the loop, by construction. The
non-losing move is to relocate: name the asymmetry as a standing condition —
"there is no available response that is read as non-hostile" — rather than
playing for a verdict the loop cannot issue. This does not break the loop; it
makes the structure visible to the observer and to any third party.

## Dynamic 3 — Threshold Veto / Accretion

**Signature.** An accommodation baseline is treated as non-revertible; a
restoration of the prior baseline is reclassified as offense; the threshold
marker is non-decreasing across cycles.

**Rules.**
- R1: A reset occurs → the accommodation baseline does not revert. (Decay
  coefficient is zero across resets.)
- R2: A restoration of the prior baseline is attempted → the attempt is
  reclassified as offense. (The only path back to reciprocity is pre-coded as a
  violation.)
- R3: A reset occurs → the active load set is the union of all prior loads. (The
  post-reset equilibrium is a stacked configuration, not a neutral baseline.)

**Loads.**
- *Standard-holding* (discrete): the observer holds the suspended standard
  without raising it, because raising it is reclassified as offense.
- *Silent-hold integral* (continuous): the accumulated standard-holding sustained
  over time; magnitude = active prohibitions × duration held. This is the
  dominant accumulating cost and the one that never appears as a discrete event.

**Falsifiers.**
- F1 (decay): the threshold marker decays across any no-initiative interval.
- F2 (autonomous): the threshold marker rises across an interval in which the
  observer took no initiative. (Falsifies the response-contingent reading.)

**Diagnostic that keeps this honest.** Genuine distress trends toward
resolution — the accommodation requirement *decays* as the state resolves. The
veto structure trends the other way — the requirement is sticky or rises,
because the suspension is functional. The diagnostic is therefore the
**derivative**, not the presence of a hold: is the requirement decaying or
accreting? A case where the requirement genuinely shrinks over time was never
this dynamic, and F1 should fire.
